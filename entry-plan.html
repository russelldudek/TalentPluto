Days 1-15

Read the system

Earn context before changing the operating cadence.

  • Shadow active accounts, founder decisions, onboarding, customization, and prototype handoffs.
  • Map customer objective, AI-search visibility problem, attribution question, CRM/data environment, decision owner, and required result.
  • Identify where account context currently lives and where it disappears.

Outputs: account brief; current-state client-to-product map; stakeholder and decision-right map; baseline for time-to-value, handoff completeness, rework, and support burden.

Days 16-30

Instrument the work

Make invisible productization decisions explicit.

  • Launch the Client Delta Register: shared core, legitimate delta, evidence gap, one-off exception.
  • Define prototype minimums: objective, user, data, authority, failure modes, evaluation, owner, and expiry.
  • Create a weekly client-to-product review with founder, product, and engineering participation.

Outputs: delta register; decision log; prototype evidence standard; client/product metric set.

Days 31-60

Prove one complete loop

  • Select one representative enterprise request with clear value and non-trivial product learning.
  • Build the smallest production-minded prototype; test data assumptions, human authority, failure behavior, and evaluation.
  • Deliver the immediate client result while capturing the reusable primitive and engineering handoff.

Outputs: working prototype; evaluation record; engineering handoff; learning returned to roadmap and next account.

Days 61-90

Make the loop repeatable

  • Convert the successful path into onboarding, customization, prototype, and handoff playbooks.
  • Prioritize productization using client value, reusability, strategic fit, support burden, and confidence.
  • Define what the role should automate, delegate, standardize, or stop doing as volume grows.

Outputs: client-to-product playbook; evidence-ranked roadmap priorities; capacity and role-boundary view; 90-day learning memo.
